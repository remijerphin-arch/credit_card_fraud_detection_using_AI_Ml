Academic Context

Dataset Name: Credit Card Fraud Detection Dataset

Dataset Context: The dataset is used to detect fraudulent credit card transactions using Machine Learning. Real-time recognition is critical to prevent cardholders from unauthorized charges.

Description: Contains transactions made by European cardholders in September 2013 over a two-day period. Due to confidentiality, the original features were PCA-transformed into features V1–V28.

Features Explanation:

  • Time: Seconds elapsed between this transaction and the first transaction in the dataset.
  • Amount: Transaction value (useful for cost-sensitive learning).
  • V1-V28: PCA principal components.
  • Class: Fraud label (0 = Genuine, 1 = Fraud).
ML Imbalance Challenge

With only 0.172% fraud transactions, standard classification accuracy is misleading (a naive model predicting all transactions as genuine achieves 99.82% accuracy).

Evaluation metrics like Precision, Recall, F1-score, ROC-AUC, and **Area Under Precision-Recall Curve (PR-AUC / AUPRC)** are used.

Model Selected: RandomForestClassifier. Ensemble decision trees are chosen for their resilience against class imbalances and outliers, and their ability to model complex non-linear partitions without overfitting.
