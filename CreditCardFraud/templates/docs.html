REST API Documentation

Endpoints and integration requirements for connecting external ESP32 micro-controllers and client systems.

POST
/api/v1/predict
Inference Gateway

Submit transaction data for real-time model prediction. Calculates risk scores and saves record to SQLite.

Request Body (JSON):
{
  "card_id": "string (Required)",
  "amount": "float (Required)",
  "merchant": "string (Required)",
  "transaction_time": "string (Required) - Format: 'YYYY-MM-DDTHH:MM:SS' or 'HH:MM:SS'",
  "location": "string (Required)",
  "device_id": "string (Required)",
  "transaction_type": "string (Required) - 'Genuine' | 'Fraud'",
  "latitude": "float/string (Required)",
  "longitude": "float/string (Required)"
}
Success Response (200 OK):
{
  "prediction": "Fraud" | "Genuine",
  "risk_score": 0.94
}
Error Responses:
  • HTTP 400 Bad Request: If missing JSON header, malformed JSON, or missing required fields.
  • HTTP 503 Service Unavailable: If model (`fraud_model.pkl`) is not trained on the system.
GET
/api/v1/transactions
Transaction Log

Retrieve logged transactions. Supports pagination using query parameters.

Query Parameters:
  • limit (int, default=100) - Maximum transactions to return.
  • offset (int, default=0) - Row offset for pagination.
Response Example:
[
  {
    "id": 1,
    "card_id": "card_8921_3920_4812",
    "amount": 85.50,
    "merchant": "Target Store",
    "location": "New York, USA",
    "device_id": "esp32_terminal_01",
    "transaction_time": "12:30:00",
    "prediction": "Genuine",
    "risk_score": 0.08,
    "timestamp": "2026-07-09 07:20:58"
  }
]
GET
/api/v1/stats
Metrics Overview

Fetch summary counts, percentages, and trend lists for charting modules.

Response Example:
{
  "total_transactions": 25,
  "today_transactions": 5,
  "total_fraud": 2,
  "today_fraud": 1,
  "total_genuine": 23,
  "today_genuine": 4,
  "avg_risk_score": 0.12,
  "fraud_percentage": 8.0,
  "hourly_transactions": { "00": 0, "01": 0, ... },
  "fraud_trend": { "2026-07-09": 1, ... },
  "genuine_trend": { "2026-07-09": 4, ... }
}
GET
/api/v1/health
Health Monitor

Verifies connectivity of server engine, SQLite database, and loading of the RandomForest pickle file.

Response Example:
{
  "server": "Running",
  "database": "Connected",
  "model": "Loaded"
}
Security Headers
Content-Type

All request payloads must be dispatched with header: Content-Type: application/json.

Scalable Authentication

The API endpoints are prepared for future JWT (JSON Web Tokens) or static API-key headers in headers for production hardware deployment.

Status Code Index
200 OK Transaction processed
400 Bad Request Malformed JSON or validation fail
401 Unauthorized Role permissions locked
503 Service Unavailable Classifier model missing
